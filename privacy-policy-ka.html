  • სახელებს, ელ-ფოსტის მისამართებს ან საკონტაქტო ინფორმაციას
  • მოწყობილობის იდენტიფიკატორებს თვალთვალის მიზნით

ლოკალური მონაცემთა შენახვა

أذان الصلاة: აზან ალ სალათ ინახავს შემდეგ მონაცემებს ადგილობრივად თქვენს მოწყობილობაზე iOS UserDefaults-ის გამოყენებით. ეს პარამეტრები ინახება მოწყობილობაზე, თუმცა მდებარეობაზე დაფუძნებულმა ფუნქციებმა შეიძლება ასევე გაგზავნოს თქვენი GPS კოორდინატები მესამე მხარის სერვისებში საჭიროების შემთხვევაში:

  • მდებარეობა: GPS კოორდინატები და ქალაქის სახელი, შენახული ლოცვის დროის გამოსათვლელად და საჩვენებლად
  • ლოცვის პარამეტრები: გამოთვლის მეთოდი, ათანის ხმები, შეტყობინებები და იქამის პარამეტრები
  • ლოცვის ქეში: 30-დღიანი ოფლაინ ლოცვის დროის ქეში
  • ყურანი: თარგმანის პარამეტრები, ქეშირებული თარგმანები, სანიშნეები და შრიფტის ზომა
  • აპლიკაციის პარამეტრები: ენა, მუქი რეჟიმი და ადაპტაციის სტატუსი
  • შესყიდვის სტატუსი: აპლიკაციაში შესყიდვების ჩანაწერი

ქსელური მოთხოვნები

ქვემოთ მოცემულია ქსელის მოთხოვნების სრული სია, რომელიც აპს შეუძლია. ამ მოთხოვნებზე არ არის მიმაგრებული პირდაპირი იდენტიფიკატორები, როგორიცაა თქვენი სახელი, ელფოსტის მისამართი ან მოწყობილობის ID:

  • Aladhan API (aladhan.com) — გამოიყენება როგორც დასაბრუნებელი მხოლოდ იმ შემთხვევაში, თუ ოფლაინ ლოცვის გაანგარიშება ვერ ხერხდება. აგზავნის თქვენს GPS კოორდინატებს და გაანგარიშების მეთოდს ლოცვის დროის გამოსათვლელად.
  • AlQuran Cloud API (alquran.cloud) — პირველი გამოყენებისას ჩამოტვირთავს ყურანის თარგმანებს. უგზავნის მხოლოდ თავის ნომერს და თარგმანის კოდს.
  • Photon / Nominatim / OpenStreetMap — გამოიყენება ერთხელ თქვენი GPS კოორდინატების ქალაქის სახელად გადასაყვანად საჩვენებლად. მდგრადი იდენტიფიკატორი არ იგზავნება.
  • DNS შემოწმება (dns.google) — სტანდარტული DNS მოთხოვნა, რომელიც გამოიყენება ფონზე ლოცვის ქეშის განახლებამდე ინტერნეტის ხელმისაწვდომობის შესამოწმებლად. აპლიკაციის მონაცემები არ იგზავნება.
